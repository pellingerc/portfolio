Iterative Design: Fetii

Part 1: Sketching

Pre-Design Thinking

Fetii is a startup that uses large sized passenger vans and buses to transport large groups of people. Similarly to Uber, users can book rides for now or later depending on their needs, but instead of one person paying with Fetii it is much easier to split the bill. Users can scan a QR code before getting on a vehicle to split the ride fare. Fetii targets a large consumer group ranging from corporate groups attending conferences to greek life groups. In this project we design a new desktop website for Fetii where users can book rides, find out about the company, and apply to work as a driver for Fetii.

Wireframes

Based on our sketches, we made wireframes based on the following design descisions

  • Website will feature both Rider and Driver functionality. Need a clear and distinct navbar that shows the user where they currently are on the page
  • Ordering a van should have a minimalist design and highlight the key items: pickup/dropoff location and time and a map for the user to visualize the route
  • Pickup Now and Pickup Later should look very similar, the user should understand what to do either way

Crit

We got lots of helpful feedback from an in-class crit. Here were our takeaways:

  • Most negative feedback and suggestions came from the “Book a Ride” page. In general, users wanted a clearer and more intuitive experience, as well as more information about the ride. Specifically, users found it unclear how to set a location, pay for the ride, specify the number of riders, and did not find an estimated price.
  • Users wanted a Profile page to keep track of their scheduled and past rides, as well as their payment information.
  • Users found the text on the website too plain and too clunky. There was a lack of general hierarchy that could help discern where to look.
  • Overall, users would have appreciated more information about the product and website. The page did not seem intuitive enough both with what the product is and how to use it.

User Testing

Tasks

Scenario: Imagine that you need to transport yourself and a group of others. Book a ride for the group. This platform is a prototype so some elements may be prefilled and not able to be interacted with.

  • Without leaving the homepage, what are your initial impressions of the website?
  • Book a ride for now
  • Submit a job application on this website
  • Set up an account on this website

Post-Test Questions

For each task,

  • Did you complete the task successfully?
  • Rate this task overall from very difficult to very easy

Analysis of Results

The first question that we asked the users was “without leaving the homepage, what are your initial impressions of the website”. All three testers mentioned the color prominence of the color purple, the name of the company, the request a ride button, sign up to drive button, and the nav bar. This is in line with what we thought users would notice on the home page of our website. One user mentioned that the items within the nav bar were not level within which was a good point and something that we had not noticed. This is something we will fix in the future. They also mentioned the idea of changing the “my account” button to be an icon instead of text. This is also another idea to consider. Another user mentioned that they thought the home page had everything they would need to book a ride.

The second task was to “book a ride for now”. Two users were able to complete the task easily. The second user was unable to complete the task successfully. They got to the final step, but never hit the confirm and pay button. This is something to consider in our design. We initially expected everyone to be able to complete this step with little difficulty, so maybe we could change the design to better indicate that this is not the final step. One user mentioned that they would like to know the estimated drop off time before the final screen. This would be a useful feature and something we should change in the future. 2 out of 3 users navigated to the book a ride feature from the nav bar, but all noticed that it was in both locations. One user pointed out that they appreciated the fact that it was linked in two locations because it made it easier to navigate through the website.

The third task was to “submit a job application on this website”. All three users were able to successfully complete this task and said that it was “very easy” to do so. All three users navigated to the application through the nav bar. They all thought that the application process was easy and straightforward. One user mentioned that they didn’t like the photo on the page that appears once the user has submitted an application. They mentioned that this photo was not diverse and representative. This was a really good point and something to consider changing in the future.

The fourth task was to “set up an account on this website”. All three users clicked on the my account tab in the nav bar. All three users were able to complete the task. One user was confused on the past drives section of the page because they just created the account. This section would presumably be blank on the actual website, but would be something to change for the prototype. One user was wondering if the past orders would be scrollable or click to open another page. This would be a design idea to consider in the future.
